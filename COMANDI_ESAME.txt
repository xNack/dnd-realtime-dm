==============================================================
  COME AVVIARE LA DEMO — DM Assistant (Real-Time D&D)
==============================================================

Prerequisito: Docker Desktop avviato.
Servono due finestre di Terminale (nuova finestra: Cmd + N).

--------------------------------------------------------------
 TERMINALE 1 — AVVIARE LA PIPELINE
--------------------------------------------------------------

  cd dnd-realtime-dm
  docker compose up

Attendere nel log queste due righe (1-2 minuti):
  dnd-spark   | [avvio] In ascolto su Kafka topic 'dnd-events'
  dnd-kibana  | ... Kibana is now available

Lasciare questo terminale aperto.

--------------------------------------------------------------
 BROWSER — APRIRE LA DASHBOARD
--------------------------------------------------------------

  http://localhost:5601
  Menu ☰ -> Dashboard -> "DM Assistant - Live"
  In alto a destra: intervallo "Last 30 minutes", auto-refresh 5s.

--------------------------------------------------------------
 TERMINALE 2 — AVVIARE LA PARTITA (replay degli eventi)
--------------------------------------------------------------

  cd dnd-realtime-dm
  SPEED=1 START_DELAY=0 ./demo-reset.sh

  (SPEED=1 = tempo reale. Valori piu' bassi = piu' lento.)
  (Se "permission denied":  bash demo-reset.sh )

Lo script ripulisce i risultati precedenti e rigioca la sessione:
i record arricchiti compaiono in Kibana in tempo reale.

--------------------------------------------------------------
 COSA OSSERVARE
--------------------------------------------------------------

- Nel Terminale 2:
    [OK] ... rischio=...    -> evento di gioco arricchito da Claude
    scartato (non e' gioco) -> frase non di gioco, filtrata
- In Kibana i record arrivano con qualche secondo di ritardo
  sull'evento: e' la latenza reale della pipeline
  (Kafka -> Spark -> Claude -> Elasticsearch -> Kibana).
- I dati sono pre-trascritti con Whisper e rigiocati in streaming
  dal producer (replay), per simulare una sessione dal vivo.

--------------------------------------------------------------
 ARRESTO
--------------------------------------------------------------

  Terminale 1: Ctrl + C
  docker compose down          (spegnimento pulito; la dashboard resta salvata)

  NB: non usare "docker compose down -v" (cancellerebbe dati e dashboard).

--------------------------------------------------------------
 IN CASO DI PROBLEMI
--------------------------------------------------------------

- Nessun record in Kibana dopo 2 minuti:
    docker compose restart logstash
- Verificare il numero di documenti salvati:
    curl "http://localhost:9200/dnd-enriched/_count?pretty"
